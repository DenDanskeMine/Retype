# Basal router konfiguration

# Første konfiguration af en router

I denne guide vil jeg tage dig igennem din første konfiguration af en cisco router.

Hvis du ikke har en router, kan du bruge Packet Tracer , som er et program som emulerer cisco udstyr.
Bemærk at du skal have en konto for at kunne downloade programmet.
Du kan også bruge Cisco DevNet, men dette er lidt mere avanceret at skulle sætte op.

Her er en simpel konfiguration af en router .

Når du starter din router op for første gang, vil du blive mødt af en prompt, som ser sådan her ud:

would you like to enter the initial configuration dialog? [yes/no]:

Her er det vigtigt at vi vælger nej, da vi selv vil konfigurere routeren.
Så du skriver bare no og trykker enter.

nu vil den prompte Promte dig med en anden prompt, som ser sådan her ud:

Press RETURN to get started!

Så du trykker selvfølgelig bare enter, og så vil du blive mødt af en prompt, som ser sådan her ud:

Router>

Når du kan se et > tegn, efterfulgt af routerens navn, er du i det der hedder user EXEC Mode , som er en begrænset tilstand, hvor du ikke kan lave de store ændringer.

# Privileged EXEC Mode

Så derfor skriver vi Enable og trykker enter, så vi kan komme ind i Privileged EXEC Mode , som er en tilstand hvor vi kan se hele routerens konfiguration, og lave små ændringer.

Når du er i Privileged EXEC Mode så ser prompten sådan her ud:

Router#

Det er typisk her vi ville lave ping, og show kommandoer.

# Global Configuration Mode

Vi vil meget gerne et tand højere op, så vi skriver configure terminal og trykker enter, så vi kan komme ind i Global Configuration Mode , som er en tilstand hvor vi kan lave ændringer i routerens konfiguration.

Når du er i Global Configuration Mode så ser prompten sådan her ud:

Router(config)#

Det er denne mode hvor vi laver alle vores ændringer, og det er også her vi vil lave vores første konfiguration.

# Hostname

Det første vi starter med, er at give routeren et navn, så vi kan identificere den.
for at gøre dette skiver vi følgende:

hostname R1

hostname er en kommando som sætter routerens navn, og R1 er det navn vi har valgt at give routeren.
hostname <navn>

Info

Bemærk at der ikke må være mellemrum i navnet.
hostname R 1
hostname R1

Når du har skrevet kommandoen, trykker du enter, og prompten vil nu se sådan her ud:

R1(config)#

# Interface mode / IP-adresse

Nu har vi givet routeren et navn, så nu vil vi gerne sætte en ip-adresse på routeren, så vi kan sætte den til at kommunikere med andre enheder.

For at gøre dette, skal vi ind i interface mode, som er en tilstand hvor vi kan lave ændringer på de forskellige porte på routeren.

For at komme ind i interface mode, skriver vi interface <navn på interface> og trykker enter.
På nogen routere kan der være forskellige interfaces, som fx FastEthernet og GigabitEthernet. Denne router har kun FastEthernet interfaces, så vi skriver interface fastethernet 0/0 og trykker enter. fastethernet er hastigheden på porten, og 0/0 er port nummeret.

Hvis du er i tvivl om hvilke interfaces der er på din router, kan du skrive show ip interface brief og trykke enter, så vil du få en liste over alle interfaces på routeren.

Info om kommandoer i forskellige modes.

Bemærk at alle show kommandoer skal skrives i Privileged EXEC Mode , og ikke i Global Configuration Mode .
Du kan skrive do foran kommandoen, så vil den virke i Global Configuration Mode .
Hvis du ikke lige magter at skrive hele kommandoen, kan du også bare skrive sh ip int br og trykke enter, så vil den også virke.

Der er rigtig mange forkortelser i ios, så det er bare med at lære dem, så du kan skrive dem hurtigt. 😄
evt se listen her: Cisco IOS Shortcuts

Når du er i interface mode, så ser prompten sådan her ud:

R1(config-if)#

Nu er vi i interface mode, og vi vil gerne sætte en ip-adresse på porten, så vi kan sætte den til at kommunikere med andre enheder.

kommandoen til dette er ip address <ip-adresse> <subnetmaske> Vi vil gerne sætte ip-adressen 172.16.0.1 på porten, så vi skriver ip address 172.16.0.1 255.255.255.0 og trykker enter.

Når du har skrevet kommandoen, trykker du enter, og prompten vil nu se sådan her ud:

R1(config-if)#

Grunden til jeg bliver ved med at skrive prompten, er for at du kan se og forstå hvilken mode du er i. 😄
Dette er meget vigtigt, at forstå, da nogen kommandoer kun kan bruges i bestemte modes.

Nu har vi sat en ip-adresse på porten, så nu vil vi gerne aktivere porten, så den kan sende og modtage data.

For at gøre dette, skriver vi no shutdown og trykker enter.

Når du har skrevet kommandoen, trykker du enter, og nu vil der komme en masse tekst på skærmen, som ser sådan her ud:

R1(config-if)# %LINK-5-CHANGED: Interface FastEthernet0/0, changed state to up

Dette er bare routeren som fortæller os at porten er blevet aktiveret / slået til.

# show running-config

Hvis du nu skriver end og trykker enter, vil du komme tilbage til Privileged EXEC Mode .

Info

exit tager dig ud af den mode du er i, og tilbage til den mode du var i før.
end tager dig tilbage til Privileged EXEC Mode .

R1#

Dette betyder at du er tilbage i Privileged EXEC Mode , og du kan nu se din konfiguration ved at skrive show running-config og trykke enter.

Du vil nu i bunden af terminalen se et promt som ser sådan her ud:

--More--

Dette betyder at der er mere tekst, som ikke kan vises på skærmen, så du skal trykke på mellemrum for at kunne se hele konfigurationen.

Man kan faktisk skrive hele konfigurationen i en tekst fil, og så kopiere og indsætte den i routeren, og så vil den konfigurere sig selv. 😄
Dog skal man huske at alt skal være skrevet rigtigt, ellers vil det selfølig ikke virke.

# Sikkerhed og SSH

Alle adgangskoder i denne guide er Cisco.

Nu er routeren konfigureret, så nu vil vi gerne sikre den, så andre ikke kan komme ind og lave ændringer på den.

For at gøre dette, skal vi lave en bruger, som har adgang til routeren, og så skal vi sætte en adgangskode på routeren. Vi skal også kryptere adgangskoden, så den ikke står i klartekst.

Der er mange muligheder for kryptering og hashing bla: Diffie-Hellman, RSA, AES, 3DES, SHA, MD5 osv.

Vi starter simplet ud, så her vil vi "bare" bruge Type 5 kryptering. Type 5 er faktisk en rigtig svær kryptering at cracke, så det er en god start.

Type 5 kryptering

Type 5 er delt op således:
$1$<salt>$<hash>

$1$ indikerer at det er Type 5 kryptering.
<salt> er en tilfældig streng som er tilføjet til passwordet, for at gøre det sværere at cracke.
$ er en seperator.
<hash> er det faktiske hashede password.

Her er passwordet i min konfig: $1$mERr$YlCkLMcTYWwkF1Ccndtll
Lad os dele det op, så du kan se hvordan det ser ud.

$1$ indikerer Type 5.
<salt> er mERr
<hash> er YlCkLMcTYWwkF1Ccndtll.

# ip domain-name

Først skal vi sætte en ip domain-name på routeren.
Dette gør vi ved at skrive ip domain-name <domæne navn> og trykke enter. inde i Global Configuration Mode .
<domæne navn> er det domæne navn som routeren er en del af - Det kan være hvad som helst.
Jeg har valgt at kalde mit domæne for test.dk.
Så jeg skriver ip domain-name test.dk og trykker enter.

Info

Bemærk at der ikke må være mellemrum i domæne navnet.
ip domain-name test .dk
ip domain-name test.dk

Domænet bliver brugt til at lave vores RSA nøgle, som vi skal bruge til at kryptere SSH sessionen.

Hvis man prøver at lave en RSA nøgle uden et domæne, vil man få denne fejl:

% Please define a domain-name first.

# crypto key generate rsa

Når vi har sat et domæne på routeren, kan vi lave en RSA nøgle.
For at lave en RSA nøgle, skriver vi crypto key generate rsa og trykker enter.
Dette vil promte os med, hvor mange bits nøglen skal være på.
Jeg har valgt at sætte den til 1024 bits, da det er en god størrelse til en router.

Info

Bemærk at jo flere bits nøglen er på, jo længere tid tager den at lave.
Jo flere bits nøglen er på, jo sværere er den at cracke.

Sådan ville det se ud når den prompter dig med spørgsmålet:

How many bits in the modulus [512]:

Her er det vigtigt at du skriver 1024 og trykker enter, så du får en 1024 bits nøgle.
Hvis du ikke skriver noget, vil den lave en 512 bits nøgle, og det er ikke særlig sikkert.
Vi skal have SSH Version 2, og det kræver en nøgle der er højrere end 768 bits.

Når du har skrevet 1024 og trykket enter, vil den prompte dig med dette:

% Generating 1024 bit RSA keys, keys will be non-exportable...[OK]

Nu har vi lavet en RSA nøgle, som vi kan bruge til at kryptere SSH sessionen.

Hvis du vil slippe for at blive promptet med spørgsmålet, kan du skrive crypto key generate rsa general-keys modulus 1024 og trykke enter.

Nu skriver vi ip ssh version 2 og trykker enter.
Nu har vi sat SSH versionen til 2.

# username og password

Det næste vi skal gøre, er at lave en bruger, som har adgang til routeren.
Vi vælger at lave en lokal bruger, man kan også sætte aaa på routeren. (Authentication, Authorization, Accounting) Men det bliver i en anden guide. 😄

For at lave en bruger, skriver vi username <brugernavn> secret <adgangskode> og trykker enter.
<brugernavn> er det brugernavn som vi vil give brugeren.
<secret> er den adgangskode som vi vil give brugeren.

Jeg har valgt at kalde min bruger for Cisco, og adgangskoden er Cisco.

Vi skal også sætte enable secret på routeren.
DEtter gør at folk også skal bruge en adgangskode, når de vil ind i Privileged EXEC Mode .

Vi skriver derfor enable secret <adgangskode>, og trykker enter.

Info

Istedet for secret kan man bruge password.

Det er dog ikke en god ide, da password ikke hasher adgangskoden, så den vil stå i klartekst i konfigurationen.

Man kan bruge service password-encryption til at gøre passwordet ikke læsbart med en Type 7 krypteret hash.

Dette kan nemt crackes, så det er kun for, at gøre det sværere at læse.

Det er derfor en god ide at bruge secret, da den bruger en stærkere MD5 hash, som er sværere at cracke.

Hvis du vil følge min konfig kan du skrive følgende:

username Cisco secret Cisco

# line vty

Nu skal vi konfigurere line vty, så vi kan logge ind på routeren via SSH.

Når vi skal konfigurere line vty (Virtual teletype), skal vi ind i Line Configuration Mode , som er en tilstand hvor vi kan lave ændringer på de forskellige virtuelle linjer på routeren.

For at komme ind i Line Configuration Mode , skriver vi line vty 0 15 og trykker enter.
0 15 er de virtuelle linjer som vi vil lave ændringer på.
0 - 15 er alle de virtuelle linjer på routeren.

Når du er i Line Configuration Mode , så ser prompten sådan her ud:

R1(config-line)#

Som standart er der transport input all på.

Det skal vi lige have ændret, så vi skriver transport input ssh og trykker enter.
Dette gør at vi kun kan logge ind på routeren via SSH, og ikke telnet.

Vi skal også fortælle routeren at vi vil bruge vores lokale bruger til at logge ind med, så vi skriver login local og trykker enter.
Vi skal også have en adgangskode på, så vi skriver password <adgangskode> og trykker enter.

Vi kommer ikke til at skulle bruge koden.
Men den er nødvendig for at vi kan logge ind på routeren, med SSH

Vi skal også ind på line con 0, så vi skriver line con 0 og trykker enter.
con står for console, og det er den fysiske port på routeren.
0 er port nummeret.

Her skal vi også sætte login local på og en adgangskode, så vi skriver login local og trykker enter.

Nu skriver vi password <adgangskode> og trykker enter.

Vi skal også ind på line aux 0, så vi skriver line aux 0 og trykker enter.
aux står for auxiliary, og det er også en fysiske port på routeren.
Den emulerer en modem forbindelse.
0 er port nummeret.

Her skal vi også sætte login local på og en adgangskode, så vi skriver login local og trykker enter.

Vi skriver også password <adgangskode> igen, og trykker enter.

# Opsummering

Nu har vi konfigureret routeren, så den er sikker, og vi kan logge ind på den via SSH.
Det sidste vi skal, er at gemme konfigurationen, så den ikke forsvinder når vi genstarter routeren.

Vi kan gøre dette på forskellige måder bla:

copy running-config startup-config

eller

write memory

Her er alt den samlede konfiguration vi har lavet på routeren:

hostname R1
!
interface fastethernet 0/0
 ip address 172.16.0.1 255.255.255.0
 no shutdown
!
ip domain-name test.dk
crypto key generate rsa general-keys modulus 1024 
ip ssh version 2
username Cisco secret Cisco
enable secret Cisco
!
line vty 0 15
 transport input ssh
 login local
 password Cisco
!
line con 0
 login local
 password Cisco
!
line aux 0
 login local
 password Cisco
